  <action>
    Create api/inference.py. Do NOT modify vocabulary.py, model/targeted_absa.py, or any config
    file — zero changes to existing ML code per D-03.

    Standard library imports: asyncio, heapq, os, time, unicodedata, pathlib.Path.
    Third-party imports: torch, torch.nn.functional as F.
    ML imports (work with PYTHONPATH=.): from vocabulary import Vocabulary;
    from model.targeted_absa import TargetedABSAModel.

    Module-level ENCODER_MAP (duplicated, NOT imported from train.py per D-03):
      "phobert_base"      -> "vinai/phobert-base"
      "phobert_large"     -> "vinai/phobert-large"
      "xlm_roberta_base"  -> "xlm-roberta-base"
      "xlm_roberta_large" -> "xlm-roberta-large"

    Module-level constant: _MAX_INFERENCE_LENGTH = 128
    (CONTEXT code_context specifies max_length=128 for inference, consistent with eval config;
    Vocabulary.tokenize_text_pair default is 256 but 128 is used here.)

    InferenceEngine.__init__(self) — no parameters; reads all config from env vars:

      Step 1 — Initialize instance attributes:
        self.ready: bool = False
        self.lock: asyncio.Lock = asyncio.Lock()

      Step 2 — Read env vars with defaults per D-01 and D-05:
        checkpoint_path = os.getenv("CHECKPOINT_PATH", "checkpoints/best_model.pt")
        data_dir        = os.getenv("DATA_DIR", "Data")
        device_str      = os.getenv("DEVICE", "")
        encoder_key     = os.getenv("MODEL_ENCODER_KEY", "phobert_large")
        attention_key   = os.getenv("MODEL_ATTENTION_KEY", "target_conditioned_attention")
        label_structure = os.getenv("MODEL_LABEL_STRUCTURE", "multitask_aspect_sentiment")

      Step 3 — Store for /models endpoint:
        self.encoder_key    = encoder_key
        self.attention_key  = attention_key
        self.label_structure = label_structure

      Step 4 — Resolve compute device:
        If device_str is non-empty: self.device = torch.device(device_str)
        Else: self.device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

      Step 5 — Validate and resolve encoder:
        If encoder_key not in ENCODER_MAP: raise ValueError with list of valid keys.
        encoder_name = ENCODER_MAP[encoder_key]
        self.encoder_name = encoder_name

      Step 6 — Build Vocabulary per D-04:
        data_files = [os.path.join(data_dir, "train.jsonl"),
                      os.path.join(data_dir, "dev.jsonl"),
                      os.path.join(data_dir, "test.jsonl")]
        Print startup message: "[InferenceEngine] Loading Vocabulary from {data_dir}/ ..."
        self.vocab = Vocabulary(data_files, encoder_name)
        Print: "[InferenceEngine] Vocabulary ready: {num_aspects} aspects, {num_sentiments} sentiments"

      Step 7 — Graceful checkpoint degradation:
        If NOT Path(checkpoint_path).exists():
          Print WARNING: "[InferenceEngine] Checkpoint not found at {checkpoint_path!r}. "
                         "Set CHECKPOINT_PATH env var or copy best_model.pt to that path. "
                         "/predict returns 503 until checkpoint is available."
          Return immediately. self.ready remains False. (Per API-06 graceful degradation.)

      Step 8 — Construct model architecture:
        Print: "[InferenceEngine] Loading {encoder_name} from {checkpoint_path} ..."
        model = TargetedABSAModel(
            encoder_name=encoder_name,
            num_aspects=self.vocab.num_aspects,
            num_sentiments=self.vocab.num_sentiments,
            label_structure_key=label_structure,
            attention_key=attention_key,
            dropout=0.2,               # matches training; model.eval() disables dropout
            classifier_hidden_size=256, # matches config/base.yaml
        )
        NOTE: num_labels is NOT passed — not required for "multitask_aspect_sentiment".

      Step 9 — Load state dict with DataParallel prefix handling:
        state_dict = torch.load(checkpoint_path, map_location=self.device, weights_only=True)
        If any key starts with "module.":
          state_dict = {k[len("module."):]: v for k, v in state_dict.items()}
        model.load_state_dict(state_dict)

      Step 10 — Finalize model:
        model.to(self.device)
        model.eval()   # CRITICAL: disables dropout layers for deterministic inference
        self.model = model
        self.ready = True
        Print: "[InferenceEngine] Model ready on {self.device}."

    InferenceEngine.predict(self, text: str, target: str) -> dict:
      Synchronous method. Called inside "async with engine.lock" in api/main.py predict route.

      Step 1 — NFC normalization per API-07:
        text   = unicodedata.normalize("NFC", text).replace("\ufeff", "")
        target = unicodedata.normalize("NFC", target).replace("\ufeff", "")

      Step 2 — Input validation per API-05 (raise ValueError with English message):
        If text.strip() == "":   raise ValueError("text must not be empty or whitespace")
        If target.strip() == "": raise ValueError("target must not be empty or whitespace")
        If target not in text:   raise ValueError("target must be an exact substring of text")

      Step 3 — Record start time: t0 = time.perf_counter()

      Step 4 — Tokenize at inference max_length:
        encoded = self.vocab.tokenize_text_pair(text, target, max_length=_MAX_INFERENCE_LENGTH)

      Step 5 — Prepare batch tensors (all unsqueeze(0) to add batch dimension):
        input_ids      = encoded["input_ids"].unsqueeze(0).to(self.device)
        attention_mask = encoded["attention_mask"].unsqueeze(0).to(self.device)
        target_mask    = encoded["target_mask"].unsqueeze(0).to(self.device)
        model_kwargs   = {"input_ids": input_ids,
                          "attention_mask": attention_mask,
                          "target_mask": target_mask}
        If "token_type_ids" in encoded (BERT-style encoders only):
          model_kwargs["token_type_ids"] = encoded["token_type_ids"].unsqueeze(0).to(self.device)

      Step 6 — Run model inference:
        with torch.no_grad():
            outputs = self.model(**model_kwargs)
        outputs dict has "aspect_logits" [1, num_aspects] and "sentiment_logits" [1, num_sentiments].

      Step 7 — Decode aspect predictions:
        aspect_probs_t = F.softmax(outputs["aspect_logits"][0], dim=-1)
        top_aspect_id  = int(aspect_probs_t.argmax().item())
        aspect_label   = self.vocab.id2aspect[top_aspect_id]
        Build all (label, score) pairs then extract top-5 using heapq.nlargest:
        all_asp_pairs = [(self.vocab.id2aspect[i], round(float(p), 4))
                         for i, p in enumerate(aspect_probs_t)]
        top5_aspects = [{"label": lbl, "score": sc}
                        for lbl, sc in heapq.nlargest(5, all_asp_pairs, key=lambda x: x[1])]

      Step 8 — Decode sentiment predictions:
        sent_probs_t    = F.softmax(outputs["sentiment_logits"][0], dim=-1)
        top_sent_id     = int(sent_probs_t.argmax().item())
        sentiment_label = self.vocab.id2sentiment[top_sent_id]
        Build all sentiment pairs, sorted descending (all 3 sentiments returned):
        all_sent_pairs = [(self.vocab.id2sentiment[i], round(float(p), 4))
                          for i, p in enumerate(sent_probs_t)]
        sentiment_probs = [{"label": lbl, "score": sc}
                           for lbl, sc in sorted(all_sent_pairs, key=lambda x: x[1], reverse=True)]

      Step 9 — Compute latency and return:
        latency_ms = int((time.perf_counter() - t0) * 1000)
        Return {
            "aspect":         aspect_label,
            "sentiment":      sentiment_label,
            "aspect_probs":   top5_aspects,
            "sentiment_probs": sentiment_probs,
            "latency_ms":     latency_ms,
        }
        (Keys match PredictResponse field names exactly for PredictResponse(**result) in main.py.)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → InferenceEngine.__init__ | Checkpoint file read at startup; path is operator-set env var, not user input |
| HTTP request → InferenceEngine.predict() | text and target are untrusted user-supplied strings |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | torch.load in __init__ | medium | mitigate | weights_only=True in torch.load (PyTorch >= 2.0 safe deserialization) prevents arbitrary code execution from a crafted checkpoint file. Checkpoint path is operator-controlled, not user-controlled. |
| T-02-02 | Information Disclosure | predict() ValueError messages | low | accept | ValueError messages are intentionally descriptive English for frontend display. They contain no internal file paths, stack traces, or environment variable values. |
| T-02-03 | Denial of Service | Vocabulary JSONL scan at startup | low | accept | Startup-time cost (~1-2 s for 3 JSONL files totaling ~6 MB). Not user-triggerable after startup. Acceptable for demo context. |
| T-02-04 | Tampering | Input Unicode manipulation | low | mitigate | NFC normalization + BOM strip in predict() Step 1 before substring validation. Pydantic max_length=2000 on text (Plan 01) prevents excessively long payloads from exhausting memory during tokenization. |
| T-02-SC | Tampering | No pip installs in this plan | low | accept | Plan 02 creates Python source files only; no package manager invocations. |
</threat_model>
