  <action>
    Make two edits to frontend/src/App.css:

    Edit 1 — In the combined `textarea, input[type="text"]` rule, change `font-size: .95rem` to
    `font-size: 1rem`. Rationale: iOS Safari triggers automatic viewport zoom when a focused
    input/textarea has a computed font-size below 16px; 1rem at default browser settings equals
    exactly 16px. Do NOT change any other property in this rule.

    Edit 2 — Append a new @media rule at the bottom of App.css (after all existing rules):
    At max-width: 768px, set .app padding to 1.5rem 0.75rem. This satisfies FE-13 which requires
    the layout to work on mobile viewports at 768px, complementing the existing 520px rules that
    handle grid columns. Keep all existing @media (max-width: 520px) rules unchanged.

    Make one edit to frontend/src/App.jsx:
    Change the h1 text from "Vietnamese ABSA Demo" to "Targeted ABSA Demo". This is the only
    change in App.jsx for this task — do not modify any other part of the file.
  </action>
