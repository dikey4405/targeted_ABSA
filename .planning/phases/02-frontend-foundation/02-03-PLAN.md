  <action>
    Create the directory frontend/src/hooks/ if it does not exist, then create
    frontend/src/hooks/usePredict.js.

    The hook uses three useState variables: result (null), isLoading (false), error (null).
    Wrap the predict function in useCallback with an empty dependency array.

    The predict(text, target) function:
    1. Sets isLoading=true, error=null, result=null.
    2. Normalizes both arguments before sending:
       - Strip BOM: text.replace(/^\uFEFF/, '').trim() — then apply .normalize('NFC')
       - Same for target: target.replace(/^\uFEFF/, '').trim().normalize('NFC')
       Store cleaned values in local const cleanText and cleanTarget.
    3. Calls fetch('/predict', ...) — the leading slash makes this a relative URL that the Vite
       dev proxy routes to http://localhost:8000/predict (per locked decision D: Vite proxy).
       The import.meta.env.VITE_API_URL prefix is intentionally empty in .env.development;
       prepend it: `${import.meta.env.VITE_API_URL ?? ''}/predict`
    4. Method: POST; headers: Content-Type application/json; body: JSON.stringify({ text:
       cleanText, target: cleanTarget }) — send ONLY these two fields, no model field (the
       API PredictRequest schema has no model field per api/schemas.py).
    5. signal: AbortSignal.timeout(15000) — 15 seconds matches PhoBERT-large CPU latency budget.
    6. If res.ok is false: parse response JSON (catch parse errors with .catch(() => ({}))), throw
       Error with body.detail ?? `Lỗi HTTP ${res.status}`.
    7. If fetch succeeds: call setResult(await res.json()).
    8. In the catch block: if err is DOMException with name 'TimeoutError', set error to
       'Model phản hồi quá lâu. Thử đoạn văn ngắn hơn.' Otherwise set error to err.message.
    9. In finally: setIsLoading(false).

    Also add a clearResult function: () => { setResult(null); setError(null); }.

    Return from the hook: { predict, result, isLoading, error, clearResult }.

    Import useState and useCallback from 'react' at the top; export the hook as a named export
    (export function usePredict) and also as default export.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser fetch → Vite proxy → FastAPI | User text and target phrase leave the browser; NFC+BOM sanitized in usePredict before the request body is constructed |
| API JSON response → PredictionResult DOM | API response values (aspect, sentiment, label strings) rendered into React JSX — auto-escaped by React, no dangerouslySetInnerHTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-04 | Injection (XSS) | PredictionResult — renders API label strings | medium | mitigate | All API string values (aspect, sentiment, label) are rendered via React JSX text nodes — React auto-escapes; dangerouslySetInnerHTML is never used; a compromised API returning malicious HTML would be rendered as escaped text |
| T-02-05 | Information Disclosure | Error messages from API 422/503/500 | low | accept | API detail strings are shown to the user — intentional UX; no internal stack traces are exposed (FastAPI sends only the detail field); appropriate for a local demo |
| T-02-06 | Denial of Service | AbortSignal.timeout(15000) | low | mitigate | 15-second timeout prevents the UI from hanging indefinitely on slow CPU inference; error message shown to user; no server-side rate limit needed for local demo |
| T-02-SC | Tampering | npm packages | high | mitigate | No new npm install in this plan; only source files created; package-lock.json unchanged |
</threat_model>
