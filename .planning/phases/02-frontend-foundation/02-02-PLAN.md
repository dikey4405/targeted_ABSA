<objective>
Extract the textarea and target-phrase input from the monolithic App.jsx into dedicated components
— ReviewInput and TargetSelector — placing them in frontend/src/components/. App.jsx is updated to
import and render the new components with identical props wiring, so user-facing behaviour is
preserved. NFC normalization is applied at the selection boundary inside ReviewInput.

Purpose: The planning context requires components to live in src/components/ and hooks in
src/hooks/. This plan creates the input layer; Plan 03 extracts the fetch layer. Separating the
textarea into ReviewInput also enables Plan 03 to keep App.jsx state management simple.

Output: Two new component files, two CSS additions to App.css, and an updated App.jsx that imports
them. Functionally equivalent to the existing monolithic App.jsx for the input path.
</objective>

  <action>
    Create the directory frontend/src/components/ if it does not exist, then create
    frontend/src/components/ReviewInput.jsx.

    The component signature accepts three props: value (string), onChange (function called with the
    new full textarea value), and onTargetSelect (function called with the trimmed selected phrase).

    Inside the component, declare a ref with useRef(null) and attach it to the textarea element.

    The selection handler (call it handleSelection) reads el.selectionStart and el.selectionEnd from
    the textarea DOM node, extracts el.value.slice(start, end), and applies:
      1. .trim() to remove leading/trailing whitespace from the selection
      2. .normalize('NFC') to canonicalize Vietnamese diacritics (prevents NFD mismatch with the API)
      3. Strip any BOM character (U+FEFF) via .replace(/^\uFEFF/, '') before calling onTargetSelect
    Only call onTargetSelect when the resulting string has length > 0 (ignore empty / whitespace-only
    selections — do not reset the target on accidental click-without-drag).

    Attach handleSelection to both onMouseUp and onKeyUp on the textarea. Wire value and onChange
    to the controlled input pattern.

    Textarea attributes to preserve from App.jsx: rows={4}, placeholder "Nhập review tiếng Việt
    vào đây...", className matching the existing App.css textarea rule (no className needed since
    the App.css textarea selector is element-level and picks up all textarea elements globally).

    Render the label "Review text" with span.hint "(bôi đen để chọn target)" as a sibling, matching
    the existing App.jsx label markup exactly so CSS stays consistent.

    Do NOT add a separate CSS file — the existing App.css element-level rules (textarea, label,
    .hint) already style this component via global selectors.
  </action>
